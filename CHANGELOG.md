# Changelog

All notable changes follow semantic versioning.

## [0.2.0] - 2026-07-16

### Added

- DreamTeam Constitution `DT-C1` and compact constitutional kernel.
- Five execution routes and M0/L1/L2/C3 code criticality.
- `offload` profile for main-model conservation.
- DCP/2 and CHP/2 with escaping rules and a Python validator.
- Symbol-level edit ownership and compact route/blueprint templates.
- Thirteen specialized Haiku workers across discovery, execution, and verification.
- Bounded-logic worker for fully specified application logic.
- Diff auditor and test-gap finder to reduce orchestrator review volume.
- `/dreamteam:doctor`.
- Configuration schema and example.
- Routing fixtures and expanded structural/protocol tests.

### Changed

- Claude Code plugin version to 0.2.0.
- `/dreamteam:run`, `review`, and `measure` for explicit ownership and v2 protocols.
- `repo-scout` and `structure-builder` replaced by narrower roles.

### Not included

- Agent Teams, nested agents, automatic hooks, remote telemetry, or autonomous worker chains.

## [0.1.0] - 2026-07-16

- Initial platform-independent core and Claude Code adapter.
