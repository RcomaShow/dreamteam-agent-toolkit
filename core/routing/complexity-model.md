# Routing Signals

DreamTeam 0.4 applies hard security, ownership, budget, verification, and calibration gates before any economic heuristic.

## Hard gates

C3 work remains executive-owned. Missing acceptance criteria, ambiguous semantics, unavailable independent review, an unregistered delegation contract, or a delegation contract comparable in complexity to the implementation select direct work or a blocked outcome.

## Measured signals

After the hard gates, evaluate:

- direct and whole-tree API-equivalent USD;
- context volume and cache cohort;
- retry and escalation probability from the calibrated task bucket;
- specification clarity and decision density;
- observed main reread ratio;
- delegation, validation, and handoff overhead;
- available runtime capabilities.

Forecasts are tied to a pricing catalog and must distinguish observed values from estimates. Insufficient calibration selects direct work unless the caller explicitly requests shadow evaluation.

## Routing outcome

Emit one route, one criticality class, a concrete primary agent role, any support/reviewer role, the ordered execution chain, selected and rejected costs, reason codes, and the active execution-policy limits.
