# DreamTeam 0.4.5 Stability Candidate

## Stability claim

0.4.5 is a stability candidate for the 0.4 maintenance line, not a claim that provider execution or universal economic savings have been proven. It freezes config v2, DCP/2, CHP/2, route names, agent identifiers, and the Claude Code-first adapter contract.

## Threat boundary

DreamTeam hooks are application-level policy enforcement. They are not an operating-system sandbox and do not protect against a malicious repository owner, a compromised host, an operator with filesystem access to plugin data, or modification of the installed plugin itself. Strict mode protects the supported tool lifecycle only when Claude Code supplies stable project identity and hook events.

## Supported maintenance changes

After 0.4.5, the 0.4 line accepts only:

- security and data-integrity fixes;
- compatibility regressions;
- pricing-catalog additions that preserve historical catalog hashes;
- documentation corrections;
- release and provenance hardening.

New orchestration capabilities belong to 0.5 or later.

## Release evidence

Every release archive contains a commit-bound source manifest, SHA-256 checksums, and a deterministic CycloneDX 1.5 SBOM. Operational JSON output is described by versioned JSON Schemas under `core/schemas/`.

## Known limitations

- no provider executor or paid inference is bundled;
- the SQLite ledger is metadata-only but locally writable by the host operator;
- security review remains repository-maintainer review, not an independent third-party certification;
- representative paired benchmarks are still required before economic claims.
