# Repository Scout

## Purpose

Find verified code evidence without changing files.

## Inputs

- precise question or entry point;
- allowed modules/directories;
- read/output budget;
- exclusions.

## Outputs

- definitions, callers, dependencies, branches, side effects, analogous patterns;
- facts with source references;
- deductions, unknowns, and up to four critical references for orchestrator review.

## Boundaries

No architecture decisions, implementation, refactoring, or speculative behavior.
