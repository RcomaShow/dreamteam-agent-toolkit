# Changelog

All notable changes follow semantic versioning.

## [0.3.0] - 2026-07-17

### Added

- Self-contained Claude Code plugin runtime and executable entry points.
- Lean and Frontier whole-tree cost forecasts with a pinned Sonnet 5-only baseline.
- Strict runtime config and four-condition Batch gating.
- Exact byte-span reread accounting, atomic SQLite budget reservation, release, reconciliation, and permits.
- Formal source-anchor parsing and protocol enforcement.
- Strict paired benchmark schema with separate quality, positive-cost, margin, and publication claims.
- Claude Code hooks for tool/subagent lifecycle enforcement.
- Deterministic adapter generation, release archives, and isolated artifact smoke tests.

### Changed

- Frontier now always accounts for Opus executive usage.
- Rejected candidates preserve audit cost while selected direct cost remains correct.
- All plugin skills, profiles, references, CI, and publication guidance use 0.3 semantics.

### Claim policy

- No universal savings claim is made.
- Billed USD and API-equivalent USD remain separate.
- Publication claims require paired quality parity, positive median and lower tail, configured margin, and minimum samples.

## [0.1.0] - 2026-07-16
