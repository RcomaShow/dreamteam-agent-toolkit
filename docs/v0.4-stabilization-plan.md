# DreamTeam 0.4 Stabilization Plan

Status: active maintenance line after 0.4.1.

## Goal

Stabilize the existing Claude Code-first architecture before starting 0.5 evaluation work. The line preserves config v2, DCP/2, CHP/2, route identifiers, agent identifiers, flat dispatch, writer/oracle separation, and zero runtime dependencies.

## Commit and release policy

Each fix version is one reviewable commit directly on top of the previous fix version:

1. **0.4.2 — Integrity Hotfix**: immutable terminal checkpoints, immutable tool/read idempotency keys, strict project-root provenance, and plugin-data path hardening.
2. **0.4.3 — Reproducibility and Schema Compatibility**: explicit pricing catalogs, effective-config hashing, ledger schema versioning, and immutable run context.
3. **0.4.4 — Verification and Portability**: benchmark arm semantics, protocol parser adversarial tests, Python-version CI matrix, and installed-wheel smoke testing.
4. **0.4.5 — Stability Candidate**: machine-readable operational schemas, deterministic SBOM generation, documented threat boundary, and frozen 0.4 maintenance policy.

## Acceptance gates for every commit

```bash
python scripts/sync_claude_adapter.py
git diff --exit-code
python scripts/validate.py
python -m unittest discover -s tests -v
python scripts/measure.py
python -m compileall dreamteam adapters/claude-code/plugins/dreamteam
python scripts/build_release.py
python scripts/smoke_plugin_artifact.py dist/dreamteam-claude-code-plugin-<version>.zip
```

## Non-goals

- no new model topology or provider executor;
- no in-memory operational ledger;
- no benchmark savings claim;
- no durable controller or additional adapter;
- no breaking protocol or configuration change.
